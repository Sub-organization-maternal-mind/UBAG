# UBAG — VPS profile (185.252.233.186)

Production sibling of `docker-compose.small.yml`: gateway, nginx-dashboard,
the live-browser session container, and the chat reaper. Dragonfly, MinIO, and
NATS are omitted; see the header comment in `../../docker-compose.vps.yml`.
The VPS is a shared box running unrelated client sites behind a single Nginx
Proxy Manager (NPM) container, so the Compose file keeps explicit service
resource limits.

**Postgres comes from the shared platform stack** (`/opt/platform` on the VPS,
one instance for every project on the box — rules in
`/opt/platform/PLATFORM-RULES.md`). The gateway joins the external `platform`
docker network; `UBAG_POSTGRES_DSN` in `deploy/vps/env.local` carries the
credentials generated by `/opt/platform/bin/provision-project.sh ubag`
(they land in `/opt/platform/projects/ubag.env`).

## One-time setup on the VPS

```sh
mkdir -p /opt/docker/ubag && cd /opt/docker/ubag
# (repo contents land here — see deploy notes)

cp deploy/vps/env.example deploy/vps/env.local
# edit deploy/vps/env.local: set UBAG_APP_SECRET to a strong random value
# (openssl rand -hex 32) and UBAG_POSTGRES_DSN to the platform credentials:
#   . /opt/platform/projects/ubag.env   # created by provision-project.sh ubag
#   postgres://$PLATFORM_PG_USER:$PLATFORM_PG_PASSWORD@platform-postgres:5432/$PLATFORM_PG_DB?sslmode=disable

mkdir -p deploy/vps/nginx-dashboard
USER_NAME=operator ./deploy/small/nginx-dashboard/set-operator-password.sh
# ^ writes deploy/small/nginx-dashboard/.htpasswd by default — copy/regenerate
# it at deploy/vps/nginx-dashboard/.htpasswd instead (the path this compose
# file mounts), or symlink. Prints the generated password ONCE — store it.

docker compose -f docker-compose.vps.yml --env-file deploy/vps/env.local up -d --build
```

## Wiring the domain (ubag.polytronx.com)

DNS already points here via Cloudflare (orange-cloud proxied) — confirmed via
`nslookup ubag.polytronx.com` resolving to Cloudflare anycast IPs, and no NPM
proxy host existed yet as of 2026-07-17 (origin returned Cloudflare `525`).

`nginx-dashboard` joins the existing `nginx-proxy-manager_default` docker
network (declared `external: true`) instead of publishing a host port — this
VPS has no active firewall (`ufw` is inactive), so every other 0.0.0.0-bound
port on the box is directly internet-reachable; joining NPM's private network
avoids adding one more.

In the NPM admin UI (`http://<vps-ip>:81` or `https://npm.polytronx.com`),
add a **Proxy Host**:
- Domain: `ubag.polytronx.com`
- Forward Hostname/IP: `ubag-nginx-dashboard`
- Forward Port: `80`
- SSL tab: request a new **Let's Encrypt** certificate, force SSL, HTTP/2 —
  matches the existing pattern already working for other `*.polytronx.com`
  entries on this box (e.g. `madni.polytronx.com`) despite Cloudflare proxying.

## Worker daemon (opt-in)

The Compose source defaults `UBAG_WORKER_DAEMON=false`, retaining one worker
process per job. Set it to `true` only when warm browser-page reuse is intended;
`UBAG_WORKER_DAEMON_SCRIPT` defaults to the bundled
`/app/apps/worker/run_worker_daemon.py`. Reused pages clear pending attachment
state before the next job.

## Scope

API + operator dashboard + live, manually authenticated browser adapters
(`UBAG_EXECUTOR_MODE=file`, `UBAG_WORKER_CONSUMER_ENABLED=true`, and
`run_live_worker.py`; `target=mock` still routes to the mock adapter). Artifacts
land on persistent local disk (`UBAG_ARTIFACT_STORE=localfs`).
