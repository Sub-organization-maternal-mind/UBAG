## Status

Accepted (2026-06-01).

## Context

A full gap analysis of the codebase against `UBAG_World_Class_Blueprint_v2.1.md`
found the repository at roughly 35–40% of the blueprint by meaningful surface — a
deliberate "v0 staged" build. Several shipped components also deviate from the
spec stack: the CLI is TypeScript (blueprint: Go + Bubble Tea), the dashboard is
vanilla JS (blueprint: SvelteKit + Skeleton + Tailwind), the gateway router is
the stdlib mux (blueprint: chi), and the database uses a parallel `gateway_*`
schema instead of the blueprint's `tenants → projects → apps → … → browser_tabs`
model with partitioned jobs and a pgvector semantic cache.

## Decision

The gap-closure program is executed under four standing constraints:

1. **Strict blueprint fidelity.** Deviating components are realigned to the exact
   spec stack — the CLI is rewritten in Go + Bubble Tea, the dashboard is rebuilt
   in SvelteKit, the router moves to chi, and the schema migrates to the full §22
   DDL. No permanent deviations are accepted; the blueprint is the contract.
2. **Dependency-ordered phasing.** Contract + schema first (Phase 1), then the
   core job → worker → result path, then periphery, then hardening.
3. **edge + small tier first.** A single-node / docker-compose system that works
   end-to-end is delivered before standard/enterprise scale-out.
4. **Live real-provider automation is a core priority.** The full stealth +
   live-execution stack (Patchright, fingerprinting, humanized input, live login)
   is built now, validated manually/staging because it is ToS-bound and not
   CI-testable. CI continues on mock/offline drivers (`UBAG_ADAPTER_OFFLINE=1`).

## Consequences

- Larger near-term churn (CLI and dashboard rewrites; schema migration) in
  exchange for a codebase that matches the document section-for-section, so every
  PR title maps to a blueprint §.
- The `gateway_*` tables are retained only as transitional views during the
  Phase 1 expand → migrate → contract migration, then removed.
- The deployment-profile system (`apps/gateway/internal/profile`) encodes the
  §4.5 feature matrix as data; tier behavior is queried, never hard-coded.
- Work is tracked as Phases 0–10 with a traceability matrix ensuring every
  blueprint section maps to a phase ("nothing left behind").
