## License posture

- SDKs, adapters, and templates: Apache-2.0.
- Server-side platform components: AGPL-3.0.

## Governance

The project starts with a BDFL model and transitions to a technical steering committee after community maturity.

## Release cadence

- Monthly patch releases.
- Quarterly minor releases.
- LTS every fourth minor with longer support.

## Supply chain

Releases should include pinned dependencies, vulnerability scans, SBOM, signed containers, signed binaries, and provenance.

## Contribution flow

Use DCO, RFCs for major changes, conventional commits, and adapter contribution guides.
