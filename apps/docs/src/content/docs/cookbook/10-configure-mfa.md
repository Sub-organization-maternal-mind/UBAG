---
title: Configure MFA
description: Require multi-factor authentication for dashboard users and API key issuance.
---

UBAG supports TOTP-based MFA (RFC 6238) for dashboard users, and hardware key enrollment
via WebAuthn for enterprise deployments.

## Enable MFA for all users

In `ubag.toml`:

```toml
[security]
mfa_required = true
mfa_methods = ["totp", "webauthn"]
mfa_grace_period_hours = 24  # time before MFA is enforced for existing users
```

## TOTP enrollment (user flow)

The dashboard walks users through TOTP setup on first login after MFA is required.
The QR code is generated by the gateway at `/v1/mfa/enroll/totp`:

```bash
curl -X POST http://localhost:8081/v1/mfa/enroll/totp \
  -H "Authorization: Bearer $SESSION_TOKEN" \
  -H "Ubag-Api-Version: 2026-05-22"
# Returns: { "qr_code_url": "otpauth://totp/...", "secret": "BASE32..." }
```

## Verify enrollment

```bash
curl -X POST http://localhost:8081/v1/mfa/verify \
  -H "Authorization: Bearer $SESSION_TOKEN" \
  -H "Ubag-Api-Version: 2026-05-22" \
  -d '{"method": "totp", "code": "123456"}'
```

## WebAuthn (hardware keys / passkeys)

```bash
# Begin registration ceremony
curl -X POST http://localhost:8081/v1/mfa/enroll/webauthn/begin \
  -H "Authorization: Bearer $SESSION_TOKEN" \
  -H "Ubag-Api-Version: 2026-05-22"

# Complete registration ceremony
curl -X POST http://localhost:8081/v1/mfa/enroll/webauthn/complete \
  -H "Authorization: Bearer $SESSION_TOKEN" \
  -H "Ubag-Api-Version: 2026-05-22" \
  -d "$WEBAUTHN_CREDENTIAL_JSON"
```

## MFA bypass for service accounts

API-key-authenticated service accounts are exempt from MFA. Only human SSO sessions
require the second factor. Configure service account exemptions:

```toml
[security]
mfa_exempt_patterns = ["svc-*"]
```

See [RBAC and ABAC](/security/rbac-abac) for role-based MFA policy configuration.
