---
title: Rotate an App Secret
description: How to safely rotate an UBAG app secret with zero downtime using the dual-secret window.
---

UBAG supports a dual-secret window so you can rotate credentials without downtime.

## Steps

1. **Generate a new secret** via the dashboard or API:

```bash
curl -X POST http://localhost:8081/v1/apps/$APP_ID/rotate-secret \
  -H "Authorization: Bearer $UBAG_APP_SECRET" \
  -H "Ubag-Api-Version: 2026-05-22" \
  -H "Idempotency-Key: $(uuidgen)"
```

Response:

```json
{
  "new_secret": "ubag_sk_...",
  "old_secret_expires_at": "2026-05-29T00:00:00Z"
}
```

2. **Update your application** to use `new_secret`. Both secrets are valid during the overlap window (default: 7 days).

3. **Verify** the new secret works by hitting `/v1/health`:

```bash
curl -H "Authorization: Bearer $NEW_UBAG_APP_SECRET" \
     -H "Ubag-Api-Version: 2026-05-22" \
     http://localhost:8081/v1/health
```

4. **Revoke the old secret** immediately if you suspect compromise:

```bash
curl -X POST http://localhost:8081/v1/apps/$APP_ID/revoke-secret \
  -H "Authorization: Bearer $NEW_UBAG_APP_SECRET" \
  -H "Ubag-Api-Version: 2026-05-22" \
  -H "Idempotency-Key: $(uuidgen)" \
  -d '{"secret_prefix": "ubag_sk_OLD..."}'
```

## TypeScript

```ts
const { newSecret, oldSecretExpiresAt } = await client.apps.rotateSecret(appId);
console.log('New secret expires old at:', oldSecretExpiresAt);
// Update your secret store / environment variable
await secretsManager.put('UBAG_APP_SECRET', newSecret);
```
